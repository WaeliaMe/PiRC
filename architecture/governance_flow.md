# Governance Flow

## Overview
Governance ensures that ecosystem participants can influence important decisions.

## Governance Process

1. Proposal Submission
A participant submits a proposal for ecosystem changes.

2. Community Discussion
The proposal is reviewed and discussed by the community.

3. Voting Phase
Token holders and contributors vote according to governance rules.

4. Execution
If quorum and approval thresholds are reached, the decision is executed.

## Governance Responsibilities

Governance may decide on:

- Ecosystem reserve usage
- Funding of new projects
- Changes to token allocation rules
- Enforcement actions against malicious actors
